# 奇遇簿 iOS App PRD

## 1. 产品概述

**产品名：奇遇簿**

奇遇簿是一款面向自然观察和旅行探索场景的 iOS App。用户可以上传图片或短视频，识别其中出现的生物或自然对象，并把每一次遇见保存为一张可回顾、可编辑、可收藏的图鉴记录。

虽然第一阶段的高频场景来自潜水中的海洋生物识别，但产品不限制在海洋。用户也可以记录鸟类、昆虫、陆地动物、植物、菌类或其他自然奇遇。

## 2. 核心理念

1. 对世界永远保持好奇和热情。
2. 不停探索新的世界，遇见新的生命。
3. 把每一次奇遇都记录下来。

## 3. 目标用户

- 潜水、徒步、观鸟、旅行、露营等自然探索爱好者。
- 经常拍到动物、植物或其他自然对象，但不知道它们是什么的人。
- 喜欢收集图鉴、手账、旅行记录和轻科普内容的人。
- 希望用 AI 辅助识别，但又不希望结果像百科词条一样生硬的人。

## 4. 产品定位

奇遇簿不是严肃物种数据库，也不是单纯相册。

它是一个“AI 识别 + 轻科普 + 私人图鉴”的记录工具：

- 识别：告诉用户“这可能是什么”。
- 科普：用普通人能理解的语言解释“它为什么有趣”。
- 记录：把这次遇见变成一张可收藏的图鉴卡。
- 回顾：让用户知道自己一路遇见过哪些生命。

## 5. MVP 范围

### 5.1 包含

- 从相册上传图片或视频。
- 视频超过 10 秒时进入片段选择页。
- AI/mock 识别后生成结构化结果。
- 识别结果自动保存到图鉴。
- 用户可编辑识别结果文案。
- 用户可编辑时间、地点、POI 或备注。
- 图鉴页以生成插画卡片为主视觉。
- 详情页展示生成图鉴图，原始照片/视频作为辅助入口。
- 详情页右上角更多菜单中支持删除记录。

### 5.2 暂不包含

- 账号系统。
- 云同步。
- 地图视图。
- 筛选和归类。
- 严格的纲目科属种图谱。
- 真实 AI 接口。
- 真正导出裁剪后的视频文件。
- 原生相机拍摄能力。

## 6. 核心用户流程

### 6.1 上传识别

1. 用户打开 App。
2. 首页显示上传入口。
3. 用户点击上传。
4. 系统打开相册选择器。
5. 用户选择图片或视频。
6. 如果是图片，直接进入识别。
7. 如果是 10 秒以内视频，直接进入识别。
8. 如果视频超过 10 秒，进入片段选择页。
9. 识别完成后进入识别结果页。
10. 结果自动保存到图鉴。

### 6.2 长视频片段选择

1. 用户选择超过 10 秒的视频。
2. 系统进入片段选择页。
3. 默认选中一个 10 秒片段。
4. 用户可以调整起点。
5. 用户确认后用该片段进行识别。

### 6.3 识别结果

识别结果页展示：

- 生成的手绘风图鉴图。
- 原始照片/视频入口。
- 时间、地点、POI 信息。
- 识别确定度。
- 中文名。
- 英文名/拉丁名/分类信息。
- 一句话判断。
- 3 条轻科普信息。

用户可以：

- 点击文字直接编辑。
- 点击地点时间旁的小笔修改元信息。
- 返回图鉴继续浏览。

### 6.4 图鉴与详情

1. 用户进入图鉴页。
2. 图鉴以卡片墙展示所有记录。
3. 每张卡片使用生成的可爱手绘图鉴图作为主视觉。
4. 用户点击卡片进入详情。
5. 详情页顶部展示生成图鉴图。
6. 右下角提供“查看原照片/原视频”入口。
7. 右上角更多菜单提供删除记录。

## 7. AI 识别输出规则

AI 必须输出结构化结果：

- `status`：明确 / 倾向判断 / 无法判断
- `confidenceLabel`：较明确 / 不确定 / 看不出
- `commonName`：中文常用名
- `englishName`：英文名，可为空
- `latinName`：拉丁名，可为空
- `taxonomy`：基础分类，可为空
- `summary`：一句话轻科普
- `features`：外形识别点
- `behavior`：习性或当前可能行为
- `observeTip`：观察建议
- `specialFact`：特别之处
- `confusion`：可能混淆对象
- `needsMoreInfo`：需要补充的照片角度或信息

### 7.1 明确识别

如果主体清晰且特征足够明显：

- 输出明确名称。
- 自动生成完整图鉴卡。
- 标注“较明确”。

### 7.2 倾向判断

如果主体可见但关键信息不足：

- 输出“可能是……”。
- 给出候选对象。
- 告诉用户需要补充哪些角度或信息。
- 仍可保存为待确认记录。

### 7.3 无法判断

如果照片太糊、主体太小、遮挡严重：

- 明确说明“看不出”。
- 不编造物种。
- 引导用户重新上传或补充更清楚的图片/视频。

## 8. 内容风格

整体科普文案风格参考用户已有的“非正经科普”笔记：

- 轻松、口语化。
- 有画面感。
- 信息密度够，但不百科腔。
- 可以有适度拟人和吐槽。
- 重点讲“怎么认出来”“它为什么有趣”“观察时注意什么”。

示例：

> 尖尖的鹰钩嘴，加上像瓦片一样叠起来的背甲，基本就是海龟界的复古穿搭选手。

## 9. 设计原则

### 9.1 平台原则

iOS App 使用 SwiftUI 原生组件思路设计：

- `NavigationStack`
- `TabView`
- `PhotosPicker`
- `Sheet`
- `Menu`
- `Form`/编辑弹窗
- `Material` 背景
- Dynamic Type 友好

### 9.2 视觉原则

- 不做复杂营销页。
- 首页直接进入上传体验。
- 图鉴卡以生成插画为主，不以原始照片为主。
- 原始照片/视频作为证据和回看入口。
- 视觉风格温暖、轻盈、像私人图鉴。
- 不局限海洋，但允许第一版样例以海龟、鹰鳐等海洋生物为主。

## 11. 数据模型

### EncounterRecord

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | String | 记录 ID |
| name | String | 中文名 |
| latin | String | 英文/拉丁/分类 |
| confidence | String | 确定度 |
| summary | String | 一句话描述 |
| facts | [Fact] | 科普条目 |
| mediaKind | image/video | 原始媒体类型 |
| mediaURL | URL? | 原始媒体路径 |
| createdAt | Date | 创建时间 |
| observedAt | String | 拍摄/遇见时间 |
| location | String | 地点/POI |
| tags | [String] | 标签 |
| trimStart | Double? | 视频片段起点 |
| trimEnd | Double? | 视频片段终点 |

### Fact

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| title | String | 条目标题 |
| text | String | 条目正文 |

## 12. 成功标准

MVP 成功标准：

- 用户可以从相册选择图片或视频。
- 长视频可以进入 10 秒片段选择页。
- 系统可以生成一条 mock 识别记录。
- 记录自动出现在图鉴里。
- 用户可以进入详情查看生成图鉴和原始媒体入口。
- 用户可以编辑识别文案和元信息。
- 用户可以删除记录。
